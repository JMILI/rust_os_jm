## 一、项目所需环境清单

### 1. 操作系统环境
- **要求**: Linux 系统（推荐 Ubuntu/Debian）
- **用途**: 作为开发主机环境
- **说明**: 虽然教程也支持 macOS 和 Windows，但 Linux 环境是最佳选择

### 2. Rust 语言环境

#### 2.1 Rust 工具链
- **rustc**: Rust 编译器（nightly 版本）
- **cargo**: Rust 包管理器（nightly 版本）
- **rustup**: Rust 工具链管理器
- **要求版本**: 至少 nightly 2020-07-15 或更新版本
- **当前推荐**: 最新 nightly 版本

#### 2.2 Rust 组件
- **rust-src**: Rust 标准库源码（用于 no_std 开发）
- **rustfmt**: 代码格式化工具
- **clippy**: 代码检查工具
- **llvm-tools-preview**: LLVM 工具链（用于链接）

#### 2.3 Rust 编译目标
- **x86_64-unknown-none**: 裸机目标（无操作系统环境）
- **说明**: 这是编译操作系统内核所需的核心目标平台

### 3. 链接器和工具链

#### 3.1 LLVM 链接器
- **lld** 或 **rust-lld**: LLVM 链接器
- **用途**: 用于链接裸机目标，替代系统默认链接器
- **说明**: 项目使用 `ld.lld` 作为链接器

#### 3.2 其他工具
- **binutils**: 二进制工具集（通常系统自带）
- **gcc**: C 编译器（某些依赖可能需要）

### 4. 构建工具

#### 4.1 bootimage
- **工具**: `bootimage` Cargo 插件
- **安装方式**: `cargo install bootimage`
- **用途**: 将编译的内核与 bootloader 组合成可启动的磁盘镜像
- **说明**: 从 post-02 开始需要此工具

#### 4.2 依赖项
- **bootloader**: Rust crate，版本 0.9+（在 Cargo.toml 中声明）

### 5. 虚拟化工具

#### 5.1 QEMU
- **工具**: `qemu-system-x86_64`
- **用途**: 运行和测试编译的操作系统镜像
- **安装方式**: 
  - Ubuntu/Debian: `sudo apt install qemu-system-x86`
  - 或 `sudo apt install qemu-system`

### 6. 可选工具（用于调试）

#### 6.1 GDB
- **工具**: GNU 调试器
- **用途**: 调试操作系统内核
- **安装方式**: `sudo apt install gdb`

#### 6.2 其他调试工具
- **objdump**: 反汇编工具（通常包含在 binutils 中）
- **readelf**: ELF 文件分析工具（通常包含在 binutils 中）

## 四、环境配置步骤

### 步骤 1: 安装 Rust 裸机目标
```bash
rustup target add x86_64-unknown-none
```

### 步骤 2: 安装 LLVM 工具
```bash
rustup component add llvm-tools-preview
```

### 步骤 3: 安装系统链接器（如果 LLVM 工具不包含）
```bash
sudo apt update
sudo apt install lld
```

### 步骤 4: 安装 bootimage 工具
```bash
cargo install bootimage
```

**注意**: 安装 bootimage 可能需要一些时间，因为它需要编译。

### 步骤 5: 安装 QEMU
```bash
sudo apt update
sudo apt install qemu-system-x86
```

### 步骤 6: 验证安装
```bash
# 检查 Rust 目标
rustup target list --installed | grep x86_64-unknown-none

# 检查 bootimage
cargo bootimage --help

# 检查 QEMU
qemu-system-x86_64 --version
```

## 五、项目配置文件说明

根据参考代码分析，blog_os 项目需要以下配置文件：

### 5.1 Cargo.toml 配置
```toml
[package]
name = "blog_os"
version = "0.1.0"
edition = "2024"

[dependencies]
bootloader = "0.9"  # 从 post-02 开始需要

[profile.dev]
panic = "abort"

[profile.release]
panic = "abort"
```

### 5.2 rust-toolchain 文件
```
nightly
```
指定使用 nightly 工具链。

### 5.3 自定义目标文件 (x86_64-blog_os.json)
项目使用自定义目标配置，包含：
- `llvm-target`: x86_64-unknown-none
- `linker`: rust-lld
- `panic-strategy`: abort
- `disable-redzone`: true
- 其他裸机特定配置

### 5.4 .cargo/config.toml (可选)
用于配置链接器和其他构建选项。

## 六、环境检查命令汇总

以下命令可用于快速检查环境状态：

```bash

# 检查系统工具
which qemu-system-x86_64

which lld

# 检查系统包
dpkg -l | grep -E "(qemu|lld|binutils)"
```
