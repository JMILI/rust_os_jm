## 🎯 概述

分配器负责管理堆内存，主要功能：
- **分配**：为请求分配内存块
- **释放**：回收不再使用的内存
- **合并**：合并相邻的空闲块

不同的分配器设计有不同的优缺点。

---

## 📖 分配器设计

### 1. 碰撞分配器（Bump Allocator）

**原理**：维护一个指针，每次分配时向前移动。

**优点**：
- 实现简单
- 分配速度快（O(1)）

**缺点**：
- 无法释放单个块
- 只能重置整个堆

**适用场景**：临时分配，可以一次性释放所有内存。

### 2. 链表分配器（Linked List Allocator）

**原理**：维护一个空闲块的链表。

**优点**：
- 可以释放单个块
- 可以合并相邻的空闲块

**缺点**：
- 分配需要遍历链表（O(n)）
- 可能产生碎片

**适用场景**：通用分配器，需要灵活的内存管理。

### 3. 固定大小块分配器（Fixed Size Block Allocator）

**原理**：为不同大小的块维护不同的链表。

**优点**：
- 分配速度快（O(1)）
- 减少碎片

**缺点**：
- 实现复杂
- 可能浪费内存（向上取整到块大小）

**适用场景**：需要高性能的场景。

## 📝 实现示例

### 碰撞分配器

```rust
pub struct BumpAllocator {
    heap_start: usize,
    heap_end: usize,
    next: usize,
    allocations: usize,
}

impl BumpAllocator {
    pub const fn new() -> Self {
        BumpAllocator {
            heap_start: 0,
            heap_end: 0,
            next: 0,
            allocations: 0,
        }
    }

    pub unsafe fn init(&mut self, heap_start: usize, heap_size: usize) {
        self.heap_start = heap_start;
        self.heap_end = heap_start + heap_size;
        self.next = heap_start;
    }
}

unsafe impl GlobalAlloc for BumpAllocator {
    unsafe fn alloc(&self, layout: Layout) -> *mut u8 {
        let alloc_start = align_up(self.next, layout.align());
        let alloc_end = match alloc_start.checked_add(layout.size()) {
            Some(end) => end,
            None => return ptr::null_mut(),
        };

        if alloc_end > self.heap_end {
            ptr::null_mut()
        } else {
            self.next = alloc_end;
            alloc_start as *mut u8
        }
    }

    unsafe fn dealloc(&self, _ptr: *mut u8, _layout: Layout) {
        // 碰撞分配器不支持释放单个块
    }
}
```

## 🔍 关键概念

### 1. 内存对齐

内存分配必须满足对齐要求，通常向上取整到对齐边界。

### 2. 内存碎片

频繁分配和释放可能产生碎片，影响内存利用率。

### 3. 分配策略

- **首次适配**：使用第一个足够大的块
- **最佳适配**：使用最小的足够大的块
- **最差适配**：使用最大的块
