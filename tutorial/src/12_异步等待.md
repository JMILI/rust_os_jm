## 🎯 概述

多任务处理允许同时运行多个任务。有两种主要方式：
- **抢占式多任务**：操作系统强制切换任务
- **协作式多任务**：任务主动让出 CPU

async/await 是 Rust 实现协作式多任务的方式。

---

## 📖 关键概念

### 1. Future Trait

```rust
pub trait Future {
    type Output;
    fn poll(self: Pin<&mut Self>, cx: &mut Context<'_>) -> Poll<Self::Output>;
}
```

- `poll`：检查 Future 是否完成
- `Poll::Ready`：Future 已完成
- `Poll::Pending`：Future 尚未完成

### 2. 状态机转换

async 函数被编译为状态机，每个 `.await` 点是一个状态。

### 3. 固定（Pinning）

某些 Future 需要固定到内存中，防止移动。

## 📝 实现步骤

### 步骤 1：创建简单的执行器

创建 `src/task/executor.rs`：

```rust
use alloc::collections::VecDeque;
use core::task::{Context, Poll, RawWaker, RawWakerVTable, Waker};

pub struct SimpleExecutor {
    task_queue: VecDeque<Task>,
}

impl SimpleExecutor {
    pub fn new() -> SimpleExecutor {
        SimpleExecutor {
            task_queue: VecDeque::new(),
        }
    }

    pub fn spawn(&mut self, task: Task) {
        self.task_queue.push_back(task)
    }

    pub fn run(&mut self) {
        while let Some(mut task) = self.task_queue.pop_front() {
            match task.poll() {
                Poll::Ready(()) => {} // 任务完成
                Poll::Pending => self.task_queue.push_back(task),
            }
        }
    }
}
```

### 步骤 2：创建异步键盘任务

```rust
use futures_util::task::AtomicWaker;

pub struct Keyboard {
    waker: AtomicWaker,
}

impl Keyboard {
    pub fn new() -> Keyboard {
        Keyboard {
            waker: AtomicWaker::new(),
        }
    }

    pub async fn read_key(&self) -> u8 {
        KeyboardScancodeStream::new().next().await
    }
}
```

## 🔍 关键概念

### 1. 协作式多任务

任务通过 `.await` 主动让出 CPU，而不是被强制切换。

### 2. Waker

Waker 用于在任务可以继续执行时唤醒它。

### 3. 执行器

执行器负责轮询 Future 并在它们准备好时运行它们。
