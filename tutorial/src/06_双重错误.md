## 📚 教程目标

本教程详细探讨双重错误异常，当 CPU 无法调用异常处理程序时会发生这种情况。通过处理此异常，我们避免了导致系统重置的致命*三重错误*。为了在所有情况下防止三重错误，我们还设置了一个*中断堆栈表*，以在单独的内核堆栈上捕获双重错误。

在本教程结束时，您将：
- 理解双重错误异常
- 实现双重错误处理程序

- 防止三重错误

---

## 🎯 概述

双重错误是一种特殊的异常，当 CPU 在处理另一个异常时出错时发生。如果双重错误也无法处理，CPU 会触发三重错误，导致系统重置。

为了防止三重错误，我们需要：
1. 实现双重错误处理程序
2. 设置中断堆栈表（IST）
3. 为双重错误分配单独的堆栈

## 📝 实现步骤

### 步骤 1：创建双重错误堆栈

在 `src/interrupts.rs` 中添加：

```rust
use x86_64::structures::gdt::{SegmentSelector, GlobalDescriptorTable, Descriptor};
use x86_64::structures::tss::TaskStateSegment;

struct Selectors {
    code_selector: SegmentSelector,
    tss_selector: SegmentSelector,
}

static mut GDT: (GlobalDescriptorTable, Selectors) = {
    let mut gdt = GlobalDescriptorTable::new();
    let code_selector = gdt.add_entry(Descriptor::kernel_code_segment());
    let tss_selector = gdt.add_entry(Descriptor::tss_segment(&TSS));
    (
        gdt,
        Selectors {
            code_selector,
            tss_selector,
        },
    )
};

static mut TSS: TaskStateSegment = TaskStateSegment::new();

const DOUBLE_FAULT_IST_INDEX: u16 = 0;

pub fn init_gdt() {
    use x86_64::instructions::segmentation::set_cs;
    use x86_64::instructions::tables::load_tss;

    unsafe {
        GDT.0.load();
        set_cs(GDT.1.code_selector);
        load_tss(GDT.1.tss_selector);
    }
}

pub fn init_idt() {
    use x86_64::structures::idt::InterruptStackFrame;

    unsafe {
        IDT.double_fault.set_handler_fn(double_fault_handler)
            .set_stack_index(DOUBLE_FAULT_IST_INDEX);
        IDT.load();
    }
}

extern "x86-interrupt" fn double_fault_handler(
    stack_frame: InterruptStackFrame,
    _error_code: u64,
) -> ! {
    panic!("EXCEPTION: DOUBLE FAULT\n{:#?}", stack_frame);
}
```

## 🔍 关键概念

### 1. 双重错误

双重错误在以下情况发生：
- 异常处理程序触发另一个异常
- 异常处理程序位于无效内存中
- 堆栈溢出导致异常处理失败

### 2. 中断堆栈表（IST）

IST 允许为特定异常指定独立的堆栈，这对于处理堆栈溢出等情况非常重要。

### 3. 任务状态段（TSS）

TSS 包含 IST 指针，用于在异常处理时切换到不同的堆栈。
