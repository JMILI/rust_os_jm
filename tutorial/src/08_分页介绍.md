## 🎯 概述

内存管理是操作系统的核心功能之一。它提供了：
- **内存隔离**：防止程序相互干扰
- **虚拟内存**：为每个程序提供独立的地址空间
- **内存保护**：防止未授权访问

x86_64 架构使用**分页**作为主要的内存管理机制。

---

## 📖 关键概念

### 1. 为什么需要内存隔离？

没有内存隔离时：
- 程序可以访问和修改其他程序的内存
- 一个程序的错误可能导致整个系统崩溃
- 无法实现多任务处理

### 2. 分段 vs 分页

#### 分段（Segmentation）
- 将内存划分为不同大小的段
- 每个段有基址和长度
- 问题：容易产生外部碎片

#### 分页（Paging）
- 将内存划分为固定大小的页（通常 4KB）
- 使用页表来映射虚拟地址到物理地址
- 优点：避免外部碎片，更灵活

### 3. 虚拟内存

虚拟内存为每个程序提供：
- **独立的地址空间**：每个程序都从地址 0 开始
- **内存保护**：可以设置读写执行权限
- **按需分页**：只在需要时分配物理内存

## 🔍 x86_64 页表结构

### 四级页表

x86_64 使用四级页表结构：

```
虚拟地址 (64位)
┌─────────┬─────────┬─────────┬─────────┬──────────┐
│  P4 (9) │  P3 (9) │  P2 (9) │  P1 (9) │ Offset(12)│
└─────────┴─────────┴─────────┴─────────┴──────────┘
    ↓        ↓        ↓        ↓          ↓
  P4表    P3表    P2表    P1表      物理地址
```

- **P4（Page Map Level 4）**：顶级页表
- **P3（Page Directory Pointer Table）**：第三级
- **P2（Page Directory）**：第二级
- **P1（Page Table）**：页表
- **Offset**：页内偏移（12位 = 4KB）

### 页表项（Page Table Entry）

每个页表项（64位）包含：
- **物理地址**：指向下一级页表或物理页
- **标志位**：
  - `Present`：页是否在内存中
  - `Writable`：是否可写
  - `User accessible`：用户空间是否可访问
  - `Execute disable`：是否禁止执行
  - 等等

---

## 💡 地址转换过程

1. CPU 将虚拟地址分解为索引和偏移
2. 使用 P4 索引查找 P4 表项
3. 使用 P3 索引查找 P3 表项
4. 使用 P2 索引查找 P2 表项
5. 使用 P1 索引查找 P1 表项
6. 使用偏移量访问物理内存
