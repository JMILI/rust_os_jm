在本教程中，我们设置可编程中断控制器，以正确地将硬件中断转发到 CPU。为了处理这些中断，我们在中断描述符表中添加新条目，就像我们为异常处理程序所做的那样。我们将学习如何获得周期性定时器中断以及如何从键盘获取输入。

---

## 🎯 概述

硬件中断是外部设备（如键盘、鼠标、定时器）通知 CPU 需要处理的方式。与异常不同，硬件中断是异步的，可以在任何时候发生。

为了处理硬件中断，我们需要：
1. 配置可编程中断控制器（PIC）
2. 在 IDT 中注册中断处理程序
3. 启用中断

## 📝 实现步骤

### 步骤 1：创建 PIC 模块

创建 `src/interrupts/pic.rs`：

```rust
use x86_64::instructions::port::Port;

pub struct Pic8259 {
    offset: u8,
    command: Port<u8>,
    data: Port<u8>,
}

impl Pic8259 {
    pub const unsafe fn new(offset: u8) -> Pic8259 {
        Pic8259 {
            offset,
            command: Port::new(0x20),
            data: Port::new(0x21),
        }
    }

    pub unsafe fn initialize(&mut self) {
        // 初始化 PIC
        self.command.write(0x11);
        self.data.write(self.offset);
        self.data.write(0x04);
        self.data.write(0x01);
    }

    pub unsafe fn mask_all(&mut self) {
        self.data.write(0xff);
    }
}

pub struct ChainedPics {
    pics: [Pic8259; 2],
}

impl ChainedPics {
    pub const unsafe fn new(offset1: u8, offset2: u8) -> ChainedPics {
        ChainedPics {
            pics: [
                Pic8259::new(offset1),
                Pic8259::new(offset2),
            ],
        }
    }

    pub unsafe fn initialize(&mut self) {
        self.pics[0].initialize();
        self.pics[1].initialize();
    }

    pub unsafe fn mask_all(&mut self) {
        for pic in &mut self.pics {
            pic.mask_all();
        }
    }
}
```

### 步骤 2：实现中断处理程序

在 `src/interrupts.rs` 中添加：

```rust
use crate::interrupts::pic::ChainedPics;

pub const PIC_1_OFFSET: u8 = 32;
pub const PIC_2_OFFSET: u8 = PIC_1_OFFSET + 8;

pub static mut PICS: ChainedPics = unsafe {
    ChainedPics::new(PIC_1_OFFSET, PIC_2_OFFSET)
};

pub fn init_idt() {
    unsafe {
        // ... 其他异常处理程序 ...
        
        IDT[InterruptIndex::Timer.as_usize()]
            .set_handler_fn(timer_interrupt_handler);
        IDT[InterruptIndex::Keyboard.as_usize()]
            .set_handler_fn(keyboard_interrupt_handler);
        
        PICS.initialize();
        IDT.load();
    }
}

extern "x86-interrupt" fn timer_interrupt_handler(
    _stack_frame: InterruptStackFrame
) {
    print!(".");
    unsafe {
        PICS.notify_end_of_interrupt(InterruptIndex::Timer.as_u8());
    }
}

extern "x86-interrupt" fn keyboard_interrupt_handler(
    _stack_frame: InterruptStackFrame
) {
    use x86_64::instructions::port::Port;

    let mut port = Port::new(0x60);
    let scancode: u8 = unsafe { port.read() };
    
    print!("{:x} ", scancode);

    unsafe {
        PICS.notify_end_of_interrupt(InterruptIndex::Keyboard.as_u8());
    }
}
```

## 🔍 关键概念

### 1. 可编程中断控制器（PIC）

PIC 负责管理硬件中断，将多个设备的中断合并到 CPU 的少数几个中断线上。

### 2. 中断向量

每个硬件中断都有一个对应的中断向量号，用于在 IDT 中查找处理程序。

### 3. 中断结束（EOI）

处理完中断后，必须向 PIC 发送 EOI 信号，否则 PIC 不会再发送该中断。
