## 📚 教程概述

本系列教程将指导您使用 Rust 编程语言创建一个小的操作系统。每个教程都是一个小步骤，包含所有需要的代码，您可以跟随学习。源代码也可以在对应的 GitHub 仓库中找到。

**最新教程：Async/Await（异步/等待）**

## 📖 教程目录

### 第一部分：Bare Bones（基础骨架）

这部分教程将带您从零开始创建一个最小的操作系统内核。

1. **[01_独立式Rust二进制程序.md](01_独立式Rust二进制程序.md)**
   - 原文：A Freestanding Rust Binary
   - 链接：https://os.phil-opp.com/freestanding-rust-binary/
   - 描述：创建操作系统内核的第一步是创建一个不链接标准库的 Rust 可执行文件。这使得可以在裸机上运行 Rust 代码，而无需底层操作系统。

2. **[02_最小Rust内核.md](02_最小Rust内核.md)**
   - 原文：A Minimal Rust Kernel
   - 链接：https://os.phil-opp.com/minimal-rust-kernel/
   - 描述：在本教程中，我们为 x86 架构创建一个最小的 64 位 Rust 内核。我们基于前一个教程的独立式 Rust 二进制程序，创建一个可启动的磁盘镜像，能够在屏幕上打印内容。

3. **[03_VGA文本模式.md](03_VGA文本模式.md)**
   - 原文：VGA Text Mode
   - 链接：https://os.phil-opp.com/vga-text-mode/
   - 描述：VGA 文本模式是一种在屏幕上打印文本的简单方法。在本教程中，我们创建一个接口，通过将所有不安全的操作封装在一个单独的模块中，使其使用安全且简单。我们还实现了对 Rust 格式化宏的支持。

4. **[04_测试.md](04_测试.md)**
   - 原文：Testing
   - 链接：https://os.phil-opp.com/testing/
   - 描述：本教程探索在 `no_std` 可执行文件中的单元测试和集成测试。我们将使用 Rust 对自定义测试框架的支持，在内核中执行测试函数。为了在 QEMU 之外报告结果，我们将使用 QEMU 和 `bootimage` 工具的不同功能。

### 第二部分：Interrupts（中断）

这部分教程将教您如何处理 CPU 异常和硬件中断。

5. **[05_CPU异常.md](05_CPU异常.md)**
   - 原文：CPU Exceptions
   - 链接：https://os.phil-opp.com/cpu-exceptions/
   - 描述：CPU 异常在各种错误情况下发生，例如访问无效内存地址或除以零时。为了响应它们，我们必须设置一个提供处理函数的*中断描述符表*。在本教程结束时，我们的内核将能够捕获断点异常并在之后恢复正常执行。

6. **[06_双重错误.md](06_双重错误.md)**
   - 原文：Double Faults
   - 链接：https://os.phil-opp.com/double-fault-exceptions/
   - 描述：本教程详细探讨双重错误异常，当 CPU 无法调用异常处理程序时会发生这种情况。通过处理此异常，我们避免了导致系统重置的致命*三重错误*。为了在所有情况下防止三重错误，我们还设置了一个*中断堆栈表*，以在单独的内核堆栈上捕获双重错误。

7. **[07_硬件中断.md](07_硬件中断.md)**
   - 原文：Hardware Interrupts
   - 链接：https://os.phil-opp.com/hardware-interrupts/
   - 描述：在本教程中，我们设置可编程中断控制器，以正确地将硬件中断转发到 CPU。为了处理这些中断，我们在中断描述符表中添加新条目，就像我们为异常处理程序所做的那样。我们将学习如何获得周期性定时器中断以及如何从键盘获取输入。

### 第三部分：Memory Management（内存管理）

这部分教程将教您如何实现内存管理功能。

8. **[08_分页介绍.md](08_分页介绍.md)**
   - 原文：Introduction to Paging
   - 链接：https://os.phil-opp.com/paging-introduction/
   - 描述：本教程介绍*分页*，这是一种非常常见的内存管理方案，我们也将用于我们的操作系统。它解释了为什么需要内存隔离，*分段*如何工作，什么是*虚拟内存*，以及分页如何解决内存碎片问题。它还探讨了 x86_64 架构上多级页表的布局。

9. **[09_分页实现.md](09_分页实现.md)**
   - 原文：Paging Implementation
   - 链接：https://os.phil-opp.com/paging-implementation/
   - 描述：本教程展示如何在内核中实现分页支持。它首先探索使物理页表帧可被内核访问的不同技术，并讨论它们各自的优缺点。然后实现地址转换函数和创建新映射的函数。

10. **[10_堆分配.md](10_堆分配.md)**
    - 原文：Heap Allocation
    - 链接：https://os.phil-opp.com/heap-allocation/
    - 描述：本教程为我们的内核添加堆分配支持。首先，它介绍动态内存，并展示借用检查器如何防止常见的分配错误。然后实现 Rust 的基本分配接口，创建堆内存区域，并设置分配器 crate。在本教程结束时，内置 `alloc` crate 的所有分配和集合类型都将可用于我们的内核。

11. **[11_分配器设计.md](11_分配器设计.md)**
    - 原文：Allocator Designs
    - 链接：https://os.phil-opp.com/allocator-designs/
    - 描述：本教程解释如何从头开始实现堆分配器。它介绍并讨论不同的分配器设计，包括碰撞分配、链表分配和固定大小块分配。对于这三种设计中的每一种，我们将创建一个可用于我们内核的基本实现。

### 第四部分：Multitasking（多任务处理）

这部分教程将教您如何实现多任务处理。

12. **[12_异步等待.md](12_异步等待.md)**
    - 原文：Async/Await
    - 链接：https://os.phil-opp.com/async-await/
    - 描述：在本教程中，我们探索*协作式多任务处理*和 Rust 的*async/await*功能。我们详细研究 async/await 在 Rust 中的工作原理，包括 `Future` trait 的设计、状态机转换和*固定*。然后，我们通过创建异步键盘任务和基本执行器，为我们的内核添加基本的 async/await 支持。

## 🛠️ 开始之前

在开始学习本教程之前，请确保您已经配置好开发环境。以下是详细的环境要求：

### 必需环境

#### 1. 操作系统环境
- **要求**: Linux 系统（推荐 Ubuntu/Debian）
- **说明**: 虽然教程也支持 macOS 和 Windows，但 Linux 环境是最佳选择
- **WSL**: Windows 用户可以使用 WSL2

#### 2. Rust 语言环境

**Rust 工具链**：
- `rustc`: Rust 编译器（**nightly 版本**）
- `cargo`: Rust 包管理器（**nightly 版本**）
- `rustup`: Rust 工具链管理器
- **要求版本**: 至少 nightly 2020-07-15 或更新版本
- **当前推荐**: 最新 nightly 版本

**Rust 组件**：
- `rust-src`: Rust 标准库源码（用于 no_std 开发）
- `rustfmt`: 代码格式化工具
- `clippy`: 代码检查工具
- `llvm-tools-preview`: LLVM 工具链（用于链接）

**Rust 编译目标**：
- `x86_64-unknown-none`: 裸机目标（无操作系统环境）
- 这是编译操作系统内核所需的核心目标平台

#### 3. 链接器和工具链

- **lld** 或 **rust-lld**: LLVM 链接器
- **binutils**: 二进制工具集（通常系统自带）
- **gcc**: C 编译器（某些依赖可能需要）

#### 4. 构建工具

- **bootimage**: Cargo 插件，用于创建可启动的磁盘镜像
  - 安装方式: `cargo install bootimage`
  - 从 post-02 开始需要此工具

#### 5. 虚拟化工具

- **QEMU**: 用于运行和测试编译的操作系统镜像
  - 安装方式: `sudo apt install qemu-system-x86` (Ubuntu/Debian)

### 环境配置步骤

#### 步骤 1: 安装 Rust nightly 工具链

```bash
# 如果还没有安装 Rust
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# 安装 nightly 工具链
rustup toolchain install nightly
rustup default nightly
```

#### 步骤 2: 安装 Rust 裸机目标

```bash
rustup target add x86_64-unknown-none
```

#### 步骤 3: 安装 LLVM 工具

```bash
rustup component add llvm-tools-preview
```

#### 步骤 4: 安装系统链接器（如果需要）

```bash
sudo apt update
sudo apt install lld
```

#### 步骤 5: 安装 bootimage 工具

```bash
cargo install bootimage
```

**注意**: 安装 bootimage 可能需要一些时间，因为它需要编译。

#### 步骤 6: 安装 QEMU

```bash
sudo apt update
sudo apt install qemu-system-x86
```

### 验证安装

使用以下命令验证环境是否正确配置：

```bash
# 检查 Rust 版本
rustc --version
cargo --version

# 检查当前工具链
rustup show

# 检查已安装的目标
rustup target list --installed

# 检查已安装的组件
rustup component list --installed

# 检查系统工具
which qemu-system-x86_64
which bootimage
which lld
```

### 环境检查清单

在开始学习之前，请确认：

- ✅ **操作系统**: Linux (Ubuntu/Debian) 或 WSL2
- ✅ **Rust 工具链**: nightly 版本已安装并激活
- ✅ **Rust 组件**: rust-src, rustfmt, clippy, llvm-tools-preview 已安装
- ✅ **Rust 编译目标**: `x86_64-unknown-none` 已安装
- ✅ **LLVM 链接器**: lld 已安装
- ✅ **bootimage 工具**: 已安装
- ✅ **QEMU**: 已安装

## 📝 学习建议

1. **按顺序学习**：教程是按照从简单到复杂的顺序设计的，建议按顺序学习
2. **动手实践**：每个教程都包含代码，建议跟随教程自己实现一遍
3. **理解原理**：不仅要会写代码，更要理解背后的原理
4. **参考源码**：遇到问题时，可以参考参考代码中的对应分支代码
   - **参考代码位置**: `../ref_code/blog_os`
   - **代码分支**: 每个教程对应一个 `post-XX` 分支（如 `post-01`, `post-02` 等）
   - **查看方式**: 使用 `git worktree` 或直接切换到对应分支
