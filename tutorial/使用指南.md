# mdbook 教程网站使用指南

## 🎉 快速开始

### 方式一：使用启动脚本（推荐）

```bash
cd tutorial
./serve.sh
```

脚本会自动：
- 构建网站（如果需要）
- 启动本地服务器
- 在浏览器中打开 http://localhost:3000

### 方式二：手动启动

```bash
cd tutorial

# 构建网站
mdbook build

# 启动服务器（可选：添加 --open 自动打开浏览器）
mdbook serve --open
```

### 方式三：直接查看

构建后的 HTML 文件位于 `book/` 目录，可以直接用浏览器打开：

```bash
# Linux
xdg-open book/index.html

# macOS
open book/index.html

# Windows
start book/index.html
```

## 📁 项目结构

```
tutorial/
├── Book.toml              # mdbook 配置文件
├── README.md              # 项目说明
├── 使用指南.md            # 本文件
├── serve.sh               # 快速启动脚本
├── .gitignore            # Git 忽略文件
├── src/                   # 源文件目录
│   ├── SUMMARY.md         # 目录结构（重要！）
│   ├── 00_教程索引.md     # 教程首页
│   ├── 01_独立式Rust二进制程序.md
│   ├── 02_最小Rust内核.md
│   ├── ...
│   └── 附录_参考资源.md
└── book/                  # 构建输出（自动生成，不要手动编辑）
    ├── index.html         # 网站首页
    ├── *.html             # 各章节 HTML 文件
    └── ...                # CSS、JS 等资源文件
```

## ✏️ 编辑内容

### 添加新章节

1. 在 `src/` 目录创建新的 Markdown 文件
2. 在 `src/SUMMARY.md` 中添加链接：

```markdown
- [新章节标题](新文件.md)
```

3. 重新构建：

```bash
mdbook build
```

### 修改现有内容

1. 直接编辑 `src/` 目录下的 Markdown 文件
2. 如果使用 `mdbook serve`，更改会自动刷新

## 🔧 常用命令

```bash
# 构建网站
mdbook build

# 启动开发服务器（支持热重载）
mdbook serve

# 启动服务器并自动打开浏览器
mdbook serve --open

# 检查链接
mdbook test

# 清理构建文件
rm -rf book/
```

## 🌐 部署到网站

### GitHub Pages

1. 构建网站：
   ```bash
   mdbook build
   ```

2. 将 `book/` 目录的内容推送到 `gh-pages` 分支

3. 在 GitHub 仓库设置中启用 GitHub Pages

### 其他静态托管

`book/` 目录包含完整的静态网站，可以部署到：
- Netlify
- Vercel
- 自己的 Web 服务器
- 任何支持静态文件的托管服务

## 🎨 自定义主题

编辑 `Book.toml` 中的 `[output.html]` 部分：

```toml
[output.html]
default-theme = "navy"  # 可选：navy, light, dark, coal, ayu
preferred-dark-theme = "navy"
```

## 📝 注意事项

1. **不要手动编辑 `book/` 目录**：这是自动生成的，每次构建都会覆盖
2. **修改目录结构**：编辑 `src/SUMMARY.md`，这是书籍的目录
3. **链接格式**：使用相对路径，mdbook 会自动处理
4. **代码块**：支持语法高亮，使用标准的 Markdown 代码块格式

## 🐛 常见问题

### Q: 构建失败怎么办？

A: 检查：
- `Book.toml` 配置是否正确
- `src/SUMMARY.md` 中的链接是否有效
- Markdown 文件语法是否正确

### Q: 链接不工作？

A: 确保：
- 使用相对路径（相对于 `src/` 目录）
- 文件名大小写正确
- 在 `SUMMARY.md` 中正确引用

### Q: 如何添加图片？

A: 
1. 在 `src/` 目录创建 `images/` 文件夹
2. 将图片放入该文件夹
3. 在 Markdown 中使用：`![描述](images/图片名.png)`
